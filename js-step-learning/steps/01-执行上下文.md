# Step 01 执行上下文

目标：理解 JavaScript 代码执行前后到底发生了什么。

## 关键点
- 全局代码会创建全局执行上下文。
- 每次函数调用会创建新的函数执行上下文。
- 执行分两阶段：创建阶段（提升）和执行阶段（逐行执行）。

## 观察代码
```js
console.log(a); // ?
console.log(sum(1, 2)); // ?

var a = 10;
function sum(x, y) {
  return x + y;
}
```

你应该得到：`a` 先是 `undefined`，函数声明可提前调用。

## 练习
1. 用自己的话解释“变量提升”和“函数提升”的差别。
2. 在练习文件中填好注释里的预期输出。

## 通关标准
- 你能画出“创建阶段 -> 执行阶段”的时间顺序。
- 你能解释为什么 `var` 是 `undefined` 而函数可调用。
